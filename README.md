# MolokoAyran 🥛

Мобильное приложение для заказа натуральных молочных продуктов от фермеров. Молоко, айран, кефир, сметана, творог, сыр и другие продукты — прямо с фермы к двери.

##  Стек технологий

- **Flutter** + **Material 3**
- **flutter_bloc** — управление состоянием
- **go_router** — навигация
- **Drift (SQLite)** — локальная БД
- **fl_chart** — графики
- **shared_preferences** — сессия и настройки
- **Equatable** — value equality
- **mocktail** + **bloc_test** — unit-тесты

##  Возможности

### Авторизация
- Регистрация и вход по email + пароль
- Онлайн-авторизация через reqres.in (демо: `eve.holt@reqres.in` / `pistol`), при отклонении — автоматический локальный fallback
- Сохранение сессии между запусками
- Каждый пользователь имеет свою корзину, заказы, адреса

### Каталог продуктов
- 14 продуктов от фермеров с категориями (молоко, айран, кефир, сметана, творог, масло, сыр, йогурт, курт, қаймақ)
- Поиск по названию
- Фильтр по категории
- Pull-to-refresh
- Skeleton loading при загрузке

### Корзина
- Добавление товаров с выбором количества
- Изменение количества прямо в корзине
- Очистка корзины с подтверждением
- **Корзина сохраняется в SQLite** — пережёёт закрытие приложения
- Badge с количеством товаров в нижней навигации

### Заказы
- Оформление заказа из корзины с выбором адреса доставки
- Список заказов с фильтрацией по статусу (Ожидает / Подтверждён / Доставлен / Отменён)
- Экран деталей заказа
- Отмена и удаление заказов

### Адреса доставки
- Создание, редактирование, удаление адресов
- Назначение адреса по умолчанию
- Выбор адреса при оформлении заказа

### Аналитика (fl_chart)
- Общая сумма расходов
- Расходы по категориям на BarChart
- Детализация в процентах

### Профиль пользователя
- Email и статистика (количество заказов, потрачено)
- Управление адресами доставки
- Переключение темы (светлая / тёмная / системная)
- Выход из аккаунта

### Дополнительно
- Онбординг при первом запуске (3 экрана)
- Светлая и тёмная темы с сохранением выбора
- Офлайн-режим: данные хранятся локально, интернет нужен только для онлайн-авторизации через reqres.in

##  Архитектура

Проект построен по принципам **Clean Architecture**:

```
lib/
├── core/                  # Общая инфраструктура
│   ├── constants/         # Константы (ключи preferences, маршруты)
│   ├── router/            # GoRouter конфигурация
│   ├── theme/             # Визуальные стили (градиенты, иконки)
│   ├── utils/             # Валидаторы
│   └── widgets/           # Общие виджеты (CustomTextField, LoadingButton)
│
├── domain/                # Бизнес-логика, не зависит от Flutter
│   ├── entities/          # User, Product, Order, Address
│   ├── repositories/      # Абстрактные интерфейсы
│   └── usecases/          # Use cases (CreateOrderUseCase, etc.)
│
├── data/                  # Реализация работы с БД
│   ├── models/            # Drift таблицы (Users, Products, Orders, CartItems, Addresses)
│   ├── datasources/       # AppDatabase + локальные источники данных
│   └── repositories/      # Реализации Repository интерфейсов
│
└── presentation/          # UI слой
    ├── blocs/             # AuthBloc, ProductBloc, OrderBloc, CartBloc, AddressBloc, ThemeBloc
    ├── screens/           # Экраны (catalog, cart, orders, analytics, profile, onboarding)
    └── widgets/           # Переиспользуемые виджеты (ProductImage, EmptyState, ProductSkeleton)
```

### База данных (5 таблиц)

| Таблица | Назначение |
|---------|------------|
| Users | Учётные записи |
| Products | Каталог продуктов |
| Orders | История заказов с адресом |
| CartItems | Корзина (отдельно для каждого пользователя) |
| Addresses | Адреса доставки |

Drift поддерживает миграции — текущая версия схемы 6.

## Запуск

```bash
# Установить зависимости
flutter pub get

# Сгенерировать код Drift
dart run build_runner build --delete-conflicting-outputs

# Запустить приложение
flutter run
```

##  Тесты

```bash
flutter test
```

В проекте **24 unit-теста**, покрывающих:
- CartBloc — добавление, удаление, обновление количества, очистка
- OrderBloc — создание, фильтрация, изменение статуса, удаление
- Use cases — CreateOrder, GetUserOrders, UpdateOrderStatus, DeleteOrder
- Сериализация OrderItem, copyWith, OrderStatus.fromString

##  Сборка APK

```bash
flutter build apk --debug
```

APK будет в `build/app/outputs/flutter-apk/app-debug.apk`

## Сценарии использования

1. **Первый запуск** → онбординг → регистрация
2. **Просмотр каталога** → поиск/фильтр → детали продукта → добавление в корзину
3. **Оформление заказа** → корзина → выбор адреса → оформление
4. **Управление заказами** → список заказов → детали → отмена/удаление
5. **Аналитика** → график расходов по категориям
6. **Настройки** → профиль → переключение темы / адреса / выход
